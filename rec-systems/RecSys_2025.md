RecSys 2025 精华速览：推荐系统的下一个十年，从这里开始
生成式 AI 的浪潮拍到了推荐系统上，这一次，浪真的很大。

每年 9 月的 RecSys，是推荐系统圈子里最期待的一次”集体对话”。今年，第 19 届 ACM 推荐系统会议（RecSys 2025）于 2025 年 9 月 22–26 日 在捷克布拉格举行。

RecSys 是 CCF 推荐目录中数据库/数据挖掘/内容检索方向的 B 类顶会，每年汇聚来自学术界（Stanford、MIT、CMU）和工业界（Pinterest、Alibaba、Netflix）的顶尖研究者。今年的议题格外密集——生成式推荐、LLM 融合、工业大规模系统、公平性……几乎每个方向都在提速。

这篇文章帮你把今年最值得关注的研究趋势梳理清楚，重点论文 + 核心思路 + 个人点评，亲测干货，建议收藏。

一、生成式推荐：推荐系统的”范式革命”来了

关键词：Chain-of-Thought / Diffusion / 生成式检索

如果要用一句话描述 2025 年推荐系统最大的变化，我会说：推荐不再是”选”，而是”生成”。

传统推荐系统的逻辑是”从候选集里召回 → 排序 → 截断”，但生成式推荐跳过了这个流程，直接让模型”生成”推荐结果，和大语言模型的工作方式如出一辙。

代表性论文：

① GRACE：基于思维链标记化的生成式推荐

GRACE: Generative Recommendation via Journey-Aware Sparse Attention on Chain-of-Thought Tokenization
核心思路：把推荐问题建模成一个序列生成任务，用 Chain-of-Thought（思维链）的方式对用户的”浏览旅程”进行标记化，再通过旅程感知的稀疏注意力机制处理多行为数据。
直白解释：就像让模型先想清楚”用户为什么喜欢这个”再给推荐，而不是直接猜。
发表于 RecSys 2025 正式论文，Pages 135–144。

② Prompt-to-Slate：用扩散模型生成推荐列表

Prompt-to-Slate: Diffusion Models for Prompt-Conditioned Slate Generation
核心思路：将 Diffusion Model 引入多物品列表（slate）生成，通过输入 prompt 来控制推荐结果的风格和内容。
为什么值得关注：这是推荐系统第一次真正意义上把”扩散范式”用到 slate 生成上，探索价值极高。

③ GenSAR：统一搜索与推荐

GenSAR: Unifying Balanced Search and Recommendation with Generative Retrieval
核心思路：用生成式检索同时处理搜索和推荐任务，打破两者的系统边界。
对于有搜推融合业务的同学，这篇很值得深读。
二、LLM × 推荐系统：从”拼积木”到”深度焊接”

关键词：零样本 / 对话推荐 / 用户模拟器 / 推理增强

LLM 在推荐系统里的应用已经不是新鲜事，但今年的研究明显在往更深处走——不再是简单地”把 LLM 当特征提取器”，而是开始做端到端的深度融合。

代表性论文：

论文	核心贡献
LLM-RecG	零样本序列推荐框架，引入语义偏差感知机制，解决冷启动
Lasso	用 LLM 构建跨域推荐的用户模拟器，生成合成行为数据
MoRE	反射器混合（Mixture of Reflectors）框架，强化 LLM 在序列推荐中的推理链
USB-Rec	提升 LLM 对话式推荐能力，让人机交互更自然

今年 LLM × 推荐的核心研究方向，总结起来是四条路：

📌 LLM 理解用户意图：不再靠 ID embedding，而是用语言模型理解用户说的是什么
📌 检索增强生成（RAG）用于推荐：动态引入外部知识，突破训练数据的局限
📌 微调 vs. RAG 的混合策略：找到最优的资源分配方式（哪些靠训练，哪些靠检索）
📌 LLM 生成合成用户行为：解决数据稀疏和冷启动的终极武器之一
三、图神经网络：老将不老，继续深耕

关键词：图对比学习 / 多模态去噪 / 冷启动

GNN 在推荐系统里虽然不是今年最亮眼的方向，但仍然贡献了一批技术扎实的工作，尤其是会话推荐（Session-based Recommendation）方向。

代表性论文：

NLGCL：提出”自然邻居层”图对比学习，不需要人工构造数据增强视图，自监督信号更自然
MDSBR：多模态去噪会话推荐，专门应对真实场景中数据嘈杂的问题
层次化图信息瓶颈：用信息论的视角处理多行为推荐，理论基础扎实
冷启动 + 图学习：通过表示学习缓解极端冷启动，工业可用性较强
💡 个人观点：GNN 在推荐系统里的角色正在从”主角”变为”基础设施”，和 LLM 联合使用的工作越来越多，未来可能是标配组合。
四、跨域推荐：知识迁移的工业潜力

关键词：LLM 蒸馏 / 认知迁移 / 公平性

跨域推荐（Cross-domain Recommendation）的核心问题是：A 平台学到的用户偏好，能不能迁移到 B 平台？ 这在工业界（尤其是多 App 产品矩阵）里有很强的落地价值。

代表性论文：

TA-DTCDR：首次引入 LLM 蒸馏的标签信息用于双目标跨域推荐，打通语义和行为两个维度
认知蒸馏增强的课程推荐：借鉴认知科学视角，用知识蒸馏打破跨领域知识边界
非重叠用户的公平跨域推荐：关注那些只在一个域有数据的用户，避免”多的更多，少的没有”
五、工业界来秀肌肉了：十亿级系统实践

关键词：长序列 / 轻量 Embedding / 基础模型

今年工业界论文的质量和数量都很亮眼，几篇来自头部公司的工作直接把工程挑战摆上台面。

论文	来自	核心内容
LONGER	蚂蚁集团	工业场景下长序列建模的大规模扩展方案，解决超长用户行为序列的计算瓶颈
PinFM	Pinterest	十亿级视觉发现平台的用户活动序列基础模型，类似推荐领域的”GPT预训练”
LEAF	-	轻量、高效、自适应的 Embedding 技术，降低大规模推荐系统的参数量
CTR 缩放定律	-	探索 CTR 模型的 Scaling Law，What works for LLM may work here too
🔥 重点关注 PinFM：Pinterest 的这篇工作尝试在推荐系统里建立”基础模型”的范式，一次训练、多场景复用，这条路如果走通了，意义不亚于 NLP 领域的 BERT。
六、多模态推荐：视觉 + 文本 + 行为的三重融合
VL-CLIP：视觉定位 + LLM 增强的 CLIP Embedding，让图文内容理解更精准，直接提升多模态推荐效果
视频 + 评论联合推荐：利用序列推荐模型同时建模视频内容和用户评论，多信号融合

多模态推荐的进入门槛在不断降低，随着 CLIP、LLaVA 等基础模型的普及，未来每家有图文内容的平台都会把多模态推荐列为标配。

七、公平性、可解释性与多利益相关方

这是今年 RecSys 另一个明显增长的方向，体现了学术界对”推荐系统的社会影响”越来越重视。

评估指标的多方共建：推荐系统的好坏，不同利益方有不同标准——平台要 GMV，用户要体验，内容方要曝光。今年有工作提出多方协同共建评估体系
算法招聘的公平性：推荐介入招聘场景下的多方公平问题，现实意义很强
总结：2025 年推荐系统的五个关键词

RecSys 2025 走完，我提炼出五个关键词，代表推荐系统接下来几年的主要方向：

#	关键词	核心变化
1	生成式范式	从”选”到”生成”，架构级重构已在路上
2	LLM 深度融合	从特征增强走向端到端联合训练
3	多模态	视觉、文本、行为三合一，门槛持续降低
4	工业大规模化	Pinterest、蚂蚁等在分享真实的十亿级实践
5	责任 AI	公平、隐私、可解释性不再是论文附录，而是核心议题

如果只能看一篇论文，我推荐 GRACE 或 PinFM——前者代表学术方向的前沿探索，后者代表工业落地的最新实践。
